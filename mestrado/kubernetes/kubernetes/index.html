    Containers e Virtualização

    System Performance Evaluation of Para Virtualization, Container Virtualization, and Full Virtualization Using Xen, OpenVZ, and XenServer

    URL: https://ieeexplore.ieee.org/abstract/document/6906035

    Introdução

    O artigo trata sobre testes feitos em três formas de virtualização: Para Virtualização, Virtualização de Containers e Virtualização complete. Os autores discorrem sobre as diferenças básicas de cada uma delas e por fim estipulam uma forma de teste de carga que será feito para verificar qual dos tipos de virtualização entrega maior performance.

    Metodologia

    Os autores utilizaram a ferramenta unixBench em um SO Debian instalado em cada uma das plataformas de virtualização. A partir da execução dos testes, eles geraram graficos para cada cenario e por fim geraram uma pontuação final para cada um dos tipos de virtualização.

    Conclusão

    Ao final do processo, os autores concluem que a Virtualização Completa usando Xen Server é a melhor, ja que apresentou a maior pontuação final e foi a mais performatica na maioria dos testes. Em alguns testes a virtualização por containers foi mais eficaz e em todos a para virtualização perdeu.

    Comentarios

    Ao observar os testes feitos pelos autores, pude observar que a virtualização completa ganha em cenarios onde o uso de recursos é compartilhado entre processos. Isto me da a entender que nos casos de microserviços ou containers simples, sem dependencias internas (apenas um processo), seria interessante usar Containers, em sistemas complexos, o melhor seria usar Virtualização Completa.

    Virtualization vs Containerization to Support PaaS

    URL: https://ieeexplore.ieee.org/abstract/document/6903537

    Introdução

    Os autores discorrem sobre as vantegens do uso de containes em PaaS. Eles discorrem sobre sabores de containers disponiveis no mercado, como LXC, Docker, Warden, lmcty e OpenVz.

    Metodologia

    Os autores fazem uma analise de cada um dos sabores informados, sem realmente tomar partido em qual seria melhor.

    Conclusão

    Os autores encerram informando que as proximas PaaS devem usar containers para conseguir atender as demandas de velocidade que desenvolvedores precisam para hospedar suas aplicações.

    Comentarios

    Observando os tipos de containers informados pelos autores, podemos observar que o lmcty da Google se uniu ao Docker no projeto Libcontainer. O OpenVZ está se fundindo com o LCX; com isso realmente as opções se tornam: LXC solo, Libcontainer (Docker, RUNC, Containerd) ou Warden.
